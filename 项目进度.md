#### 3. 例程 IDF v5.5.1 适配 (`14-handheld(new)/`)

原始例程基于 IDF v5.4.3，已完成以下迁移工作：

##### I2C 驱动迁移（核心改动）
旧版 I2C 驱动 (`driver/i2c.h`) 与新版 (`driver/i2c_master.h`) 不能在同一 I2C 端口混用。涉及改动：

- **`esp32_s3_szp.h`**: `#include "driver/i2c.h"` → `#include "driver/i2c_master.h"`
- **`esp32_s3_szp.c` - `bsp_i2c_init()`**: 用 `i2c_new_master_bus()` 创建总线，再用 `i2c_master_bus_add_device()` 为 QMI8658、PCA9557 创建设备句柄
- **QMI8658/PCA9557 读写**: `i2c_master_write_read_device()` / `i2c_master_write_to_device()` → `i2c_master_transmit_receive()` / `i2c_master_transmit()`（使用设备句柄）
- **触摸屏初始化**: 去掉 `(esp_lcd_i2c_bus_handle_t)` 类型转换，让 C11 `_Generic` 宏正确选择 v2 I2C 驱动；添加 `scl_speed_hz = 100000`
- **音频编解码器**: ES8311/ES7210 的 I2C 配置从 `.port = BSP_I2C_NUM` 改为 `.bus_handle = i2c_bus_handle`

##### sdkconfig.defaults 适配
- 中文注释替换为英文（IDF 的 kconfgen 使用 GBK 编码，无法读取 UTF-8 中文）
- 添加 `CONFIG_I2C_SKIP_LEGACY_CONFLICT_CHECK=y`（IDF 的 esp_lcd 组件同时编译 v1/v2 I2C 驱动，需跳过冲突检查）

##### 依赖清理
- 从 `idf_component.yml` 移除 `espressif/esp-sr: "~1.6.0"`（项目未使用语音识别功能，esp-sr 的传递依赖 esp-dsp 会触发 GCC 内部编译器错误 Segmentation Fault）

### 阶段二：板级适配

**状态: 进行中**

#### 背景分析

esp-claw 使用 `esp_board_manager` 组件管理板级定义，通过 3 个 YAML 文件自动生成配置代码：
- `board_info.yaml` — 板子身份（名称、芯片、厂商）
- `board_peripherals.yaml` — 总线定义（I2C/SPI/I2S/RMT/GPIO/LEDC 引脚和参数）
- `board_devices.yaml` — 设备列表（类型、驱动芯片、配置、依赖、关联总线）

加上 `sdkconfig.defaults.board`（板级 sdkconfig）和 `setup_device.c`（自定义设备工厂函数）。

esp-claw 现有 3 个板级定义：
1. `esp32_S3_DevKitC_1` — 最小（仅 LED 灯带）
2. `esp32_S3_DevKitC_1_breadboard` — USB 摄像头 + USB 音频 + SPI LCD
3. `m5stack_cores3` — I2C 编解码器 + SPI LCD + I2C 触摸（最接近我们的板子）

#### 参考板对比 (m5stack_cores3 vs SZP)

| 外设 | SZP（我们） | m5stack_cores3（参考） | 匹配度 |
|------|------------|----------------------|--------|
| IO 扩展器 | PCA9557 (I2C 0x19) | AW9523B (I2C) | 类似，需自定义工厂 |
| LCD | ST7789 (SPI, 320x240) | ILI9341 (SPI, 320x240) | 类似，需 LCD 面板工厂 |
| 触摸 | FT5x06 (I2C) | FT5x06 (I2C) | 完全相同 |
| DAC | ES8311 (I2C+I2S) | AW88298 (I2C+I2S) | 类似，编解码器类型 |
| ADC | ES7210 (I2C+I2S) | ES7210 (I2C+I2S) | 完全相同 |
| 摄像头 | GC0308 (DVP) | 无 | 我们多 |
| IMU | QMI8658 (I2C) | 无 | 我们多 |
| SD 卡 | SDMMC 1-bit | 无 | 我们多 |

## 踩坑记录

### 1. VS Code ESP-IDF 插件配置

插件存储配置在三个层级，全部需要正确配置才能工作：
- **workspace settings**: `.vscode/settings.json`（项目级）
- **global settings**: `AppData/Roaming/Code/User/settings.json`（用户级）
- **内部缓存**: `AppData/Roaming/Code/User/workspaceStorage/<hash>/state.vscdb`（SQLite 数据库）

其中 state.vscdb 的 `idf.currentSetup` 字段会缓存旧路径，导致芯片目标选择功能异常。需要用 sqlite3 直接修改数据库才能修复。

### 2. sdkconfig.defaults 编码问题

IDF 的 Python 工具链（kconfgen）在中文 Windows 上使用 GBK 编码读取配置文件。UTF-8 中文注释会导致 `UnicodeDecodeError: 'gbk' codec can't decode byte`。解决方案：注释全部使用英文。

### 3. I2C 新旧驱动冲突

ESP-IDF v5.5.x 中，旧 I2C 驱动 (`driver/i2c.h`) 和新 I2C 驱动 (`driver/i2c_master.h`) 不能在同一端口混用。旧驱动源码中有 `__attribute__((constructor))` 函数，在程序启动时自动检测并 abort。

此外，IDF 的 `esp_lcd` 组件总是同时编译 v1（旧 API）和 v2（新 API）两个 LCD I2C 面板驱动。即使只用 v2，v1 也会把旧 I2C 驱动链入二进制。需要设置 `CONFIG_I2C_SKIP_LEGACY_CONFLICT_CHECK=y` 跳过检查。

### 4. _Generic 宏类型匹配

`esp_lcd_new_panel_io_i2c()` 在 C 中用 `_Generic` 宏根据 bus 参数类型选择 v1 或 v2。如果将 `i2c_master_bus_handle_t`（指针类型）强制转换为 `esp_lcd_i2c_bus_handle_t`（= `uint32_t`），`_Generic` 会匹配到 `default` 分支即 v1（旧 API），导致运行时 I2C 冲突。正确做法是直接传 `i2c_master_bus_handle_t` 类型的变量。

### 5. esp-dsp GCC 内部编译器错误

`espressif/esp-sr` 组件的传递依赖 `esp-dsp` 在编译 `dspi_conv_f32_ansi.c` 时触发 GCC ICE（Internal Compiler Error: Segmentation Fault）。esp-sr 在本项目中未使用（已验证所有源文件无任何 esp-sr 相关 #include 或 API 调用），移除后编译通过。
